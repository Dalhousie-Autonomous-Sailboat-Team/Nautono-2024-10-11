\doxysubsection{Detailed Description}
This service provides a thread-\/safe, buffered debug print facility via UART for Free\+RTOS-\/based applications.

It can also be used without Free\+RTOS, but will then not be thread-\/safe nor have efficient waiting when the buffer is full.

The following peripherals are used by this module\+:
\begin{DoxyItemize}
\item SERCOM (Serial Communication Interface)
\end{DoxyItemize}

The outline of this documentation is as follows\+:

\item \mbox{\hyperlink{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_examples}{Examples}}
\item \mbox{\hyperlink{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_api_overview}{API Overview}}
\end{DoxyItemize}\hypertarget{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_prerequisites}{}\doxysubsection{Prerequisites}\label{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_prerequisites}
The reader is assumed to be familiar with configuration of the SERCOM USART driver since this service uses similar configuration settings.

To enable Free\+RTOS-\/support, the symbol {\ttfamily \+\_\+\+\_\+\+FREERTOS\+\_\+\+\_\+} must be defined. This will typically be done automatically if Free\+RTOS is included from ASF. As this service is meant to be used in tasks, the Free\+RTOS scheduler should be started before any of the print functions are called.

To get thread-\/safety and efficient waiting with other OS or schedulers, the following functions in \mbox{\hyperlink{dbg__print_8c}{dbg\+\_\+print.\+c}} must be modified\+:
\begin{DoxyItemize}
\item \+\_\+dbg\+\_\+wait\+\_\+for\+\_\+lock()
\item \+\_\+dbg\+\_\+unlock()
\item \+\_\+dbg\+\_\+interrupt\+\_\+handler()
\item \+\_\+dbg\+\_\+wait\+\_\+for\+\_\+requested\+\_\+space()
\item \mbox{\hyperlink{group__asfdoc__common__freertos__dbg__print__group_ga9bd1ce061869e93a4bb5438aa599d048}{dbg\+\_\+init()}}
\end{DoxyItemize}\hypertarget{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_module_overview}{}\doxysubsection{Module Overview}\label{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_module_overview}
This service essentially consists of a print buffer and functions for doing protected writes to it, and an interrupt handler which performs transmission of the buffer contents using a SERCOM module in UART mode. It has been written for use in Free\+RTOS-\/based applications where concurrent tasks need to output debug messages, for example via the Communication Device Class (CDC) of the Embedded Debugger (EDBG) on Xplained Pro boards.

This service can be used without Free\+RTOS, but will then not be thread safe nor have efficient waiting when the buffer is full. To support other OSes and schedulers, see \mbox{\hyperlink{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_prerequisites}{Prerequisites}} for a list of the functions that must be modified.

This service should not be used within interrupts.\hypertarget{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_buffer}{}\doxysubsubsection{Print Buffer}\label{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_buffer}
The reason for using a print buffer is to minimize the time it takes for a task to output its debug message, so the impact on application response time and functionality is reduced. Instead of waiting for the debug message to be sent, the calling task only needs to wait for the message to get into the print buffer. Of course, if the print buffer becomes full, the tasks must wait for buffer space to free up.

Both buffer size and UART baud rate are configurable, so one may find a reasonable balance between memory usage and interrupt rate. As a minimum requirement, the print buffer should not be smaller than the longest debug message in the application.\hypertarget{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_access}{}\doxysubsubsection{Access Control}\label{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_access}
Access control for the print buffer is built into this service to prevent concurrent calls from corrupting each other\textquotesingle{}s print output. The Free\+RTOS mechanism for this access control, a mutex, will keep other calling tasks waiting while the print buffer is being modified by one task. Once the print buffer becomes available again, the waiting task with the highest priority is given access to the buffer.

Note that this mechanism may cause tasks of lower priority to be kept waiting indefinitely if the print buffer is small and there is a lot of waiting for buffer space to free up. In such cases, the higher priority tasks may end up taking turns waiting for enough buffer space to get their respective debug messages into it.\hypertarget{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_config}{}\doxysubsubsection{Configuration}\label{group__asfdoc__common__freertos__dbg__print__group_asfdoc_common_freertos_dbg_print_config}
This service implements a single debug print interface, which is compile-\/time configured via the header file \mbox{\hyperlink{conf__dbg__print_8h}{conf\+\_\+dbg\+\_\+print.\+h}} to ensure the smallest size code and highest execution speed.

{\footnotesize\ttfamily enum \mbox{\hyperlink{group__group__sam0__utils__status__codes_ga751c892e5a46b8e7d282085a5a5bf151}{status\+\_\+code}} dbg\+\_\+init (\begin{DoxyParamCaption}\item[{void}]{ }\end{DoxyParamCaption})}



Initialize and enable debug UART. 

This function sets up the configured SERCOM with the following static features\+:
\begin{DoxyItemize}
\item asynchronous, internally clocked (UART)
\item TX only
\item 1 stop bit
\item 8-\/bit data
\item LSB first
\item no parity bit
\end{DoxyItemize}

The baud rate, SERCOM signal MUX and pin MUX are set up as configured in conf\+\_\+debug\+\_\+print.\+h, which also contains the configuration of which SERCOM to use.

The SERCOM UART is left enabled after this function call.

\begin{DoxyReturn}{Returns}
Indication whether or not initialization succeeded. 
\end{DoxyReturn}

{\em STATUS\+\_\+\+OK} & if initialization was successful. \\
\hline

\hline
